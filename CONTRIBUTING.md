# Contributing

Thanks for helping improve Stelyph.

## Basics

- **Bugs / ideas:** open an issue. For security problems, do **not** open an
  issue — see [SECURITY.md](SECURITY.md).
- **Code:** fork, branch, open a pull request against `main`. PRs need review
  before merge.
- **Build & test** before submitting:

  ```sh
  cargo build
  cargo test
  cargo fmt --check
  cargo clippy
  ```

- Keep commits focused; commit messages follow the `type: summary` style used
  in the log (`feat:`, `fix:`, `docs:`, `chore:`).
